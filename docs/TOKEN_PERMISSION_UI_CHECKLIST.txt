# 토큰 권한 UI 회귀 체크리스트

## 목적
Firebase Rules 강화 이후, GM이 플레이어에게 토큰 관련 권한을 안전하게 부여/회수할 수 있는지 확인한다.

## 확정 기준
- 기본적으로 GM/owner가 토큰을 관리한다.
- 일반 플레이어는 권한을 받은 만큼만 토큰 관련 행동을 할 수 있다.
- 토큰 권한은 아래 3개로 나뉜다.
  - 토큰 이동: moveToken
  - 토큰 생성: createToken
  - 토큰 편집: editToken
- `토큰 권한 부여` 버튼은 위 3개만 부여한다.
- `토큰 권한 회수` 버튼은 위 3개만 회수한다.
- `전체 권한 부여`는 토큰 외에도 manageMap / manageBgm / sendDesc까지 부여하므로 신중하게 사용한다.
- BGM 기능은 아직 미구축이므로 manageBgm은 실제 기능 테스트 보류 상태다.
- 맵/장면 전환 관련 권한은 아직 추가 안정화가 필요한 영역이므로, manageMap을 부여하더라도 별도 테스트가 필요하다.

## 테스트 순서
1. GM 계정으로 방 설정 창을 연다.
2. 플레이어 카드에 `토큰 권한 부여`, `토큰 권한 회수`, `전체 권한 부여`, `전체 권한 회수` 버튼이 보이는지 확인한다.
3. `토큰 권한 부여` 클릭 후 플레이어에게 아래 권한만 켜졌는지 확인한다.
   - 토큰 이동
   - 토큰 생성
   - 토큰 편집
4. 플레이어 계정에서 `+ 캐릭터 토큰`, `+ 패널 토큰` 버튼이 보이는지 확인한다.
5. 플레이어 계정에서 토큰 생성/이동/편집이 가능한지 확인한다.
6. GM 계정에서 `토큰 권한 회수` 클릭 후 플레이어의 토큰 버튼이 사라지는지 확인한다.
7. 플레이어 계정에서 토큰 이동/편집 시도가 막히는지 확인한다.
8. `전체 권한 부여`는 토큰 외 권한까지 켜지는지 확인만 하고, 필요 없으면 즉시 `전체 권한 회수` 또는 개별 회수한다.

## 회귀 주의
- 권한 없는 플레이어에게 토큰 생성 버튼이 보이면 안 된다.
- 권한 없는 플레이어가 토큰 이동/편집/삭제를 실제 DB에 저장할 수 있으면 안 된다.
- GM은 항상 토큰 생성/이동/편집/삭제가 가능해야 한다.
- 권한 변경 후 새로고침 없이도 UI가 갱신되는지 확인한다.

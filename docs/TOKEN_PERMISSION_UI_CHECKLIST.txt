# 토큰 권한 UI 회귀 체크리스트

## 목적
Firebase Rules 강화 이후, GM이 플레이어에게 토큰/맵 관련 권한을 안전하게 부여/회수할 수 있는지 확인한다.

## 확정 기준
- 기본적으로 GM/owner가 토큰과 맵세팅을 관리한다.
- 토큰 공개/비공개 visibility 변경은 GM만 가능하다.
- 일반 플레이어는 권한을 받은 만큼만 토큰/맵 관련 행동을 할 수 있다.
- 토큰 권한은 아래 3개로 나뉜다.
  - 토큰 이동: moveToken
  - 토큰 생성: createToken
  - 토큰 편집: editToken
- 맵 관리 권한은 manageMap이다.
  - manageMap을 부여하면 `+ 맵세팅`과 맵세팅 레이어 버튼이 표시된다.
  - manageMap 권한자는 맵세팅 ZIP 적용, 맵세팅 레이어 표시/숨김/순서변경/삭제를 사용할 수 있다.
  - 장면 전환은 계속 GM 전용이다.
- `토큰 권한 부여` / `토큰 권한 회수` 빠른 버튼은 사용하지 않는다.
- 토큰/맵 권한은 개별 권한 버튼으로 조정한다.
- `전체 권한 부여`는 토큰 외에도 manageMap / manageBgm / sendDesc까지 부여하므로 신중하게 사용한다.
- BGM 기능은 구축 완료 상태이며, manageBgm 권한자는 GM과 동일하게 BGM을 조작할 수 있다.

## 테스트 순서
1. GM 계정으로 방 설정 창을 연다.
2. 플레이어 카드에 `토큰 권한 부여`, `토큰 권한 회수` 버튼이 보이지 않는지 확인한다.
3. 개별 권한 버튼으로 아래 토큰 권한을 각각 켜고 끌 수 있는지 확인한다.
   - 토큰 이동
   - 토큰 생성
   - 토큰 편집
4. createToken 권한을 켜면 플레이어 계정에서 `+ 캐릭터 토큰`, `+ 패널 토큰` 버튼이 보이는지 확인한다.
5. 플레이어 계정에서 토큰 생성/이동/편집 권한이 각각 개별 권한대로 작동하는지 확인한다.
6. manageMap 권한만 켰을 때 플레이어 계정에서 `+ 맵세팅`과 맵세팅 레이어 버튼이 보이는지 확인한다.
7. manageMap 권한자가 맵세팅 ZIP 적용, 맵세팅 레이어 숨김/표시/순서변경/삭제를 사용할 수 있는지 확인한다.
8. manageMap만 켜도 `+ 장면 전환`은 보이지 않는지 확인한다.
8-1. BGM 관리 권한을 켜면 플레이어가 BGM 추가/삭제/재생/이전/다음/반복/seek/플레이리스트 순서 변경을 사용할 수 있는지 확인한다.
8-2. BGM 관리 권한을 회수하면 플레이어에게 BGM 조작 UI가 즉시 사라지거나 비활성화되는지 확인한다.
9. GM 계정에서 개별 토큰/맵 권한을 회수한 후 플레이어의 버튼/조작이 막히는지 확인한다.
10. `전체 권한 부여`는 토큰 외 권한까지 켜지는지 확인만 하고, 필요 없으면 즉시 `전체 권한 회수` 또는 개별 회수한다.
11. GM이 토큰을 비공개로 저장하면 권한 없는 플레이어 화면에서 토큰/상태패널/편집창 접근이 차단되는지 확인한다.
12. 플레이어가 토큰 편집 권한을 가지고 있어도 비공개 토큰은 이동/편집/삭제/스탠딩 변경할 수 없는지 확인한다.

## 회귀 주의
- 권한 없는 플레이어에게 토큰 생성 버튼과 맵세팅 버튼이 보이면 안 된다.
- 권한 없는 플레이어가 토큰 이동/편집/삭제를 실제 DB에 저장할 수 있으면 안 된다.
- 플레이어가 visibility 값을 직접 바꾸거나 비공개 토큰을 조작할 수 있으면 안 된다.
- manageMap 권한 없는 플레이어가 맵세팅 ZIP 적용/레이어 조작을 할 수 있으면 안 된다.
- manageMap 권한자는 importedMapObject 패널 토큰 생성/수정/삭제가 가능해야 한다.
- GM은 항상 토큰 생성/이동/편집/삭제와 맵세팅 조작이 가능해야 한다.
- 권한 변경 후 새로고침 없이도 UI가 갱신되는지 확인한다.

# 토큰/맵 권한 UI 회귀 체크리스트

## 목적
Firebase Rules 강화 이후, 권한 없는 플레이어에게 위험한 조작 버튼이 노출되지 않고, GM이 부여한 권한만큼만 UI와 실제 동작이 열리는지 확인한다.

## 확정 기준
- 기본적으로 GM/owner가 토큰과 맵을 관리한다.
- 일반 플레이어는 권한을 받은 만큼만 관련 행동을 할 수 있다.
- 토큰 권한은 아래 3개로 나뉜다.
  - 토큰 이동: moveToken
  - 토큰 생성: createToken
  - 토큰 편집: editToken
- `토큰 권한 부여` / `토큰 권한 회수` 빠른 버튼은 사용하지 않는다.
- 토큰 권한은 개별 권한 버튼으로 조정한다.
- `전체 권한 부여`는 토큰 외에도 manageMap / manageBgm / sendDesc까지 부여하므로 신중하게 사용한다.
- BGM 기능은 아직 미구축이므로 manageBgm은 실제 기능 테스트 보류 상태다.
- 장면 전환은 GM 전용이다. manageMap 권한을 받아도 장면 전환 설정창은 열리지 않아야 한다.

## 권한별 UI 기준
- createToken이 없으면 `+ 캐릭터 토큰`, `+ 패널 토큰` 버튼이 보이지 않아야 한다.
- editToken이 없으면 삭제/편집 계열 조작이 실제 저장되지 않아야 한다.
- moveToken이 없으면 토큰 드래그 이동이 저장되지 않아야 한다.
- manageMap만 단독으로 있어도 `+ 맵세팅`은 보이지 않는 것이 정상이다.
  - 맵세팅 ZIP 적용은 imported panel token 생성/편집을 동반하므로 `manageMap + createToken + editToken`이 필요하다.
- 맵세팅 레이어 창은 `manageMap + editToken`이 있을 때만 보여야 한다.
  - 레이어 숨김/순서/삭제는 imported panel token 상태 변경을 동반할 수 있다.
- `+ 장면 전환`은 GM에게만 보여야 한다.
- BGM 추가/관리 버튼은 manageBgm 권한자에게만 보여야 한다.
- desc 버튼은 sendDesc 권한자에게만 보여야 한다.

## 테스트 순서
1. GM 계정으로 방 설정 창을 연다.
2. 플레이어 카드에 `토큰 권한 부여`, `토큰 권한 회수` 버튼이 보이지 않는지 확인한다.
3. 개별 권한 버튼으로 아래 권한을 각각 켜고 끌 수 있는지 확인한다.
   - 토큰 이동
   - 토큰 생성
   - 토큰 편집
   - 맵 관리
   - BGM 관리
   - desc 입력
4. 권한 없는 플레이어 계정에서 아래 버튼이 보이지 않는지 확인한다.
   - `+ 캐릭터 토큰`
   - `+ 패널 토큰`
   - `+ 맵세팅`
   - 맵세팅 레이어 버튼
   - `+ 장면 전환`
   - `+ BGM`
   - desc 버튼
5. createToken을 켜면 `+ 캐릭터 토큰`, `+ 패널 토큰` 버튼이 보이는지 확인한다.
6. createToken을 끄면 두 토큰 생성 버튼이 다시 사라지는지 확인한다.
7. manageMap만 켜도 `+ 맵세팅`이 보이지 않는지 확인한다.
8. manageMap + createToken + editToken을 모두 켜면 `+ 맵세팅`이 보이는지 확인한다.
9. manageMap + editToken을 켜면 맵세팅 레이어 버튼이 보이는지 확인한다.
10. GM이 모든 권한을 회수하면 플레이어 UI가 다시 제한 상태로 돌아오는지 확인한다.

## 회귀 주의
- 권한 없는 플레이어에게 토큰 생성 버튼이 보이면 안 된다.
- 권한 없는 플레이어가 토큰 이동/편집/삭제를 실제 DB에 저장할 수 있으면 안 된다.
- 권한 없는 플레이어에게 맵세팅 버튼이 보이면 안 된다.
- manageMap 단독 부여만으로 맵세팅 ZIP 적용 버튼을 보여주면 안 된다.
- GM은 항상 토큰 생성/이동/편집/삭제, 맵세팅, 레이어, 장면 전환이 가능해야 한다.
- 권한 변경 후 새로고침 없이도 UI가 갱신되는지 확인한다.

ITC — CHAT / POPOUT REGRESSION CHECKLIST
=========================================

목적
----
채팅/팝아웃 관련 수정 후 아래 항목을 확인한다.
특히 popout.js는 기본창과 별도의 렌더 경로를 가지므로, desc / image / speak-as / history paging 회귀가 쉽게 발생할 수 있다.

절대 기준
---------
1. 기본창과 팝아웃은 같은 메시지 데이터를 같은 의미로 렌더해야 한다.
2. 팝아웃에서 이미지 URL이 일반 텍스트 채팅처럼 노출되면 안 된다.
3. 팝아웃에서 desc 지문이 일반 채팅처럼 보이면 안 된다.
4. speak-as 스탠딩 변경 명령어 @이름 / @(이름)은 화면 표시 텍스트에 노출되면 안 된다.
5. 이전 채팅 로딩은 기본창과 팝아웃 모두에서 유지되어야 한다.

A. desc / dsec 지문 렌더
-----------------------
테스트:
1. 기본창에서 desc 모드로 메시지를 보낸다.
2. 팝아웃을 열고 같은 메시지를 확인한다.
3. 팝아웃에서 desc 모드로 메시지를 보낸다.
4. 기본창과 팝아웃 양쪽 표시를 비교한다.

정상 기준:
- 기본창과 팝아웃 모두 지문 형태로 보인다.
- 이름, 아바타, 시간 영역이 일반 채팅처럼 붙지 않는다.
- 흰색 계열 글자다.
- 볼드 느낌이 유지된다.
- 기울기 효과가 없어야 한다.
- 위아래 가로줄/border가 없어야 한다.

주의:
- desc / dsec / msg-dsec는 모두 desc 계열로 정규화되어야 한다.
- 과거 호환 때문에 dsec를 읽을 수는 있어도, 새 메시지는 desc로 저장하는 것이 맞다.

B. 텍스트 꾸미기
---------------
테스트:
1. 기본창에서 아래 메시지를 보낸다.
   - *굵게 테스트*
   - **기울기 테스트**
   - ***굵게기울기 테스트***
2. 팝아웃에서도 같은 형태로 보이는지 확인한다.
3. 팝아웃에서 같은 메시지를 보내고 기본창 표시도 확인한다.

정상 기준:
- 기본창과 팝아웃에서 꾸미기 결과가 같아야 한다.
- 원문 * 기호가 불필요하게 그대로 보이면 안 된다.
- 줄바꿈 표시도 양쪽에서 동일해야 한다.

C. 일반 이미지 메시지
--------------------
테스트:
1. 기본창에서 이미지를 1장 전송한다.
2. 팝아웃에서 이미지가 표시되는지 확인한다.
3. 이미지가 URL 문자열 채팅처럼 보이지 않는지 확인한다.
4. 팝아웃 이미지를 클릭해 확대 표시가 가능한지 확인한다.

정상 기준:
- 이미지가 실제 이미지로 보여야 한다.
- URL 원문이 일반 채팅 텍스트로 노출되면 실패다.
- 이미지 로드 실패가 발생해도 일반 텍스트 메시지 형태로 떨어지면 안 된다.

D. 이미지 옵션
-------------
테스트:
1. 가로폭 채우기 옵션을 켜고 이미지를 보낸다.
2. 닉네임/시간 숨김 옵션이 있다면 켜고 이미지를 보낸다.
3. 기본창과 팝아웃 표시를 비교한다.

정상 기준:
- 가로폭 채우기 표시가 팝아웃에서도 반영된다.
- 닉네임/시간 숨김 상태가 팝아웃에서도 반영된다.
- 이미지 정렬/여백이 기본창과 크게 다르면 추가 점검한다.

E. speak-as 이미지 / 아바타 / 이름색
------------------------------------
테스트:
1. speak-as 저널을 선택한다.
2. 일반 텍스트 메시지를 보낸다.
3. 이미지를 보낸다.
4. 팝아웃에서 이름, 아바타, nameColor가 저널 기준인지 확인한다.

정상 기준:
- 계정 오너 이름/아바타로 잘못 표시되면 실패다.
- 이미지도 speak-as 저널 이름/아바타/nameColor 기준을 유지해야 한다.

F. speak-as 스탠딩 변경 @ 명령어 숨김
------------------------------------
테스트:
1. speak-as 저널을 선택한다.
2. @스탠딩이름 대사내용 형태로 메시지를 보낸다.
3. 필요하면 @(스탠딩이름) 대사내용 형태도 확인한다.
4. 기본창과 팝아웃 표시를 비교한다.

정상 기준:
- 화면에 @스탠딩이름 또는 @(스탠딩이름)이 보이면 실패다.
- 스탠딩 변경 자체는 기존처럼 작동해야 한다.
- 실제 대사 내용은 유지되어야 한다.

G. 이전 채팅 로딩
----------------
테스트:
1. 채팅이 충분히 쌓인 방에서 기본창 채팅 스크롤을 맨 위로 올린다.
2. 이전 메시지가 추가로 보이는지 확인한다.
3. 팝아웃에서도 스크롤을 맨 위로 올린다.
4. 이전 메시지가 추가로 보이는지 확인한다.

정상 기준:
- 위로 스크롤하면 이전 메시지가 prepend되어야 한다.
- 불러온 이전 메시지가 즉시 trim되어 사라지면 실패다.
- 스크롤 위치가 갑자기 맨 아래로 튀면 추가 점검한다.
- 새 메시지 수신 시 하단 자동 스크롤은 기존처럼 유지되어야 한다.

H. DM 채널
---------
테스트:
1. GM이 DM 채널을 만든다.
2. 기본창에서 DM 메시지 전송을 확인한다.
3. 팝아웃에서 같은 DM 채널 메시지를 확인한다.
4. DM에서 이미지 / desc / speak-as / 이전 채팅 로딩을 확인한다.

정상 기준:
- 전체 채팅과 DM 채팅이 섞이면 실패다.
- 팝아웃 DM 버튼과 unread dot이 사라지면 실패다.
- DM 이미지가 URL 텍스트로 보이면 실패다.

I. 잡담 탭
---------
테스트:
1. 기본창 잡담 탭에서 메시지를 보낸다.
2. 팝아웃 잡담 탭에서 반영되는지 확인한다.
3. 팝아웃을 닫았다 다시 열어도 잡담 기록이 유지되는지 확인한다.

정상 기준:
- 팝아웃 잡담은 DOM 추출 fallback보다 메시지 스냅샷 기반 동기화를 우선해야 한다.
- 최근 메시지만 보이고 이전 메시지가 누락되는 경우 추가 점검한다.

J. 팝아웃 열기/닫기 반복
-----------------------
테스트:
1. 팝아웃을 연다.
2. 일반 채팅 / desc / 이미지 / speak-as 메시지를 확인한다.
3. 팝아웃을 닫고 다시 연다.
4. 같은 항목을 다시 확인한다.

정상 기준:
- 닫았다 열어도 desc 스타일이 회귀하면 안 된다.
- 이미지가 URL 텍스트로 회귀하면 안 된다.
- 팝아웃 상단 탭과 DM 버튼이 사라지면 실패다.

수정 시 주의사항
---------------
- popout.js에서 메시지 렌더 분기 순서를 흔들지 말 것.
- 권장 분기 순서:
  1. desc 계열
  2. image / speak-as-image 계열
  3. system 계열
  4. dice 계열
  5. normal 계열
- 이미지 메시지를 normal 렌더 fallback으로 보내지 말 것.
- desc 스타일을 별도 italic/border 스타일로 되돌리지 말 것.
- 기본창 DOM에서 보이는 HTML을 복사하는 방식에만 의존하지 말 것.
- 가능한 경우 메시지 객체 스냅샷을 기준으로 팝아웃을 다시 렌더할 것.

최근 안정화 기준
---------------
- 팝아웃 렌더 안정화 v1: desc / image / normal 타입 정규화 및 image fallback 보강.
- 팝아웃 동기화 안정화 v2: getChatRenderSnapshot 기반 메시지 전달 구조 보강.
- speak-as @ 스탠딩 명령어는 팝아웃 화면 표시 텍스트에서도 숨김 처리.

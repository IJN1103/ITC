Firebase Rules 롤백 가이드
==========================

목적
----
Firebase Console Rules 강화 적용 후 문제가 생겼을 때 즉시 이전 Rules로 되돌리는 절차를 고정한다.

현재 파일 기준
--------------
- `database.rules.json`: 현재 운영 적용 Rules.
- `firebase-rules/database.rules.rollback-current.json`: 하드닝 적용 전 기존 Rules 백업.

롤백해야 하는 경우
-----------------
- 방 생성/입장/재입장이 막힘
- 일반 채팅/잡담/DM 전송이 막힘
- GM이 방 종료/삭제를 할 수 없음
- GM이 맵세팅 ZIP을 업로드할 수 없음
- GM이 장면 전환 저장/전환을 할 수 없음
- GM이 캐릭터 토큰/패널 토큰을 생성/편집/삭제할 수 없음
- 팝아웃 채팅이 권한 오류로 동기화되지 않음
- 콘솔에 PERMISSION_DENIED가 반복 발생하고 원인 파악이 어렵다

롤백 순서
--------
1. `firebase-rules/database.rules.rollback-current.json`을 연다.
2. 파일 전체 내용을 복사한다.
3. Firebase Console > Realtime Database > Rules로 이동한다.
4. 현재 Rules 전체를 지우고 복사한 내용을 붙여넣는다.
5. Publish 한다.
6. GM/플레이어 양쪽 브라우저를 모두 새로고침한다.
7. 아래 최소 기능을 확인한다.
   - 방 입장
   - 채팅 전송
   - GM 방 삭제
   - 토큰 생성
   - 맵세팅 ZIP 업로드
   - 장면 전환 저장

주의
----
- 롤백 후에도 클라이언트 캐시나 열린 탭 상태 때문에 잠시 오류가 남을 수 있다.
- 롤백 후에는 GM/플레이어 양쪽 브라우저를 모두 새로고침한다.
- 문제가 해결되면 강화 Rules를 다시 바로 적용하지 말고, 어떤 경로에서 막혔는지 먼저 기록한다.
- 하드닝 Rules가 다시 필요하면 `database.rules.json`을 기준으로 수정한다.

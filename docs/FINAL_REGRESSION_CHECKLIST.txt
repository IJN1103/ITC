ITC — FINAL REGRESSION CHECKLIST
================================

목적
----
최근 안정화된 패널 토큰 / 맵세팅 / 장면 전환 / 채팅 / 팝아웃 기능이 서로 회귀하지 않는지 최종 확인한다.
새 기능 개발 전 또는 큰 패치 후 이 문서를 기준으로 한 번 점검한다.

진행 원칙
---------
- 한 번에 전부 고치지 말고, 실패한 항목만 좁혀서 수정한다.
- 채팅/팝아웃 문제는 docs/CHAT_POPOUT_REGRESSION_CHECKLIST.txt를 우선 기준으로 삼는다.
- 맵세팅 문제는 docs/MAPSETTING_REGRESSION_CHECKLIST.txt를 우선 기준으로 삼는다.
- 패널 토큰 문제는 docs/PANEL_TOKEN_CORE_REGRESSION_CHECKLIST.txt를 우선 기준으로 삼는다.
- Firebase Rules / Cloudinary / Vercel 설정 변경이 필요하면 코드 수정 전에 사용자에게 먼저 알린다.

A. 채팅 / 팝아웃
---------------
1. 기본창에서 일반 채팅 전송 → 팝아웃에도 동일하게 표시된다.
2. 기본창에서 desc 전송 → 팝아웃에서도 흰색 볼드 지문, 기울기 없음, 위아래 줄 없음.
3. 팝아웃에서 desc 전송 → 기본창/팝아웃 모두 지문 형태로 표시된다.
4. *굵게*, **기울기**, ***굵게기울기***가 기본창/팝아웃에서 동일하게 보인다.
5. 기본창에서 이미지 전송 → 팝아웃에서 URL 텍스트가 아니라 이미지로 보인다.
6. speak-as 이미지 전송 → 팝아웃에서 저널 이름 / 아바타 / nameColor가 유지된다.
7. @스탠딩이름 또는 @(스탠딩이름) 명령어는 기본창/팝아웃 표시 텍스트에 노출되지 않는다.
8. 기본창 맨 위 스크롤 → 이전 메시지가 추가 로딩된다.
9. 팝아웃 맨 위 스크롤 → 이전 메시지가 추가 로딩된다.
10. DM 채널에서도 desc / 이미지 / 이전 메시지 로딩이 유지된다.
11. 잡담 탭 메시지가 팝아웃에 동기화된다.
12. 팝아웃을 닫았다 다시 열어도 위 항목들이 유지된다.

B. 패널 토큰
------------
1. +패널 토큰 생성 시 캐릭터 토큰이 아니라 패널 토큰으로 생성된다.
2. 우클릭 편집 시 패널 토큰 전용 편집창이 열린다.
3. 앞면 이미지 저장 후 맵에 표시된다.
4. 뒷면 이미지 저장 후 앞/뒷면 전환이 된다.
5. 클릭 액션 `채팅 보내기`가 정상 작동한다.
6. 클릭 액션 `매크로`가 정상 작동한다.
7. speak-as 선택 상태에서 클릭 액션이 저널 기준으로 발화된다.
8. 이름을 완전히 비우면 맵 위 이름 라벨이 표시되지 않는다.
9. 새로고침 후에도 패널 토큰 상태가 유지된다.

C. 맵세팅 ZIP / 레이어
---------------------
1. 맵세팅 ZIP 업로드 시 배경 이미지가 적용된다.
2. object item은 imported panel token으로 생성된다.
3. imported panel token은 캐릭터 토큰 편집창으로 열리지 않는다.
4. 위치고정 imported panel token 위 좌클릭 드래그 시 토스트 없이 맵 팬 이동이 된다.
5. 위치고정 imported panel token의 앞/뒷면 전환과 편집창 기능은 유지된다.
6. 레이어 눈 아이콘으로 숨김/표시 시 실제 맵 위 imported panel token도 함께 반영된다.
7. 레이어 순서 변경 시 실제 겹침 순서도 바뀐다.
8. 단일 레이어 삭제 시 연결된 imported panel token이 함께 삭제된다.
9. 전체 삭제 시 오브젝트 레이어, 연결된 imported panel token, 배경 이미지가 함께 삭제된다.
10. 배경만 남아 있어도 전체 삭제 버튼이 보여야 한다.

D. 장면 전환
------------
1. 장면 전환 설정창이 열린다.
2. 씬 카드가 가로 16:9 카드형으로 표시된다.
3. 씬 카드 드래그 순서 변경이 가능하다.
4. 창을 닫았다 열어도 씬 순서가 유지된다.
5. 새로고침 후에도 씬 순서가 유지된다.
6. 씬 우클릭 복제가 가능하다.
7. 복제 씬은 원본 바로 뒤에 생성된다.
8. 씬 더블클릭 시 LIVE 씬이 실제 맵에 반영된다.
9. 씬 A에 맵세팅 ZIP 적용 후 저장 → 씬 B 전환 → 씬 A 복귀 시 배경/오브젝트/레이어 상태가 유지된다.
10. 씬 A에서 전체 삭제 후 저장 → 씬 B 전환 → 씬 A 복귀 시 삭제 상태가 유지된다.
11. 새로고침 후 active 씬 상태가 유지된다.

E. 토큰 좌표 / 맵 조작
---------------------
1. 일반 캐릭터 토큰 드래그가 정상 작동한다.
2. 일반 패널 토큰 드래그가 정상 작동한다.
3. 줌인/줌아웃해도 토큰 간 거리가 변하지 않는다.
4. 창 크기를 바꿔도 토큰 위치와 거리가 유지된다.
5. 위치고정 맵세팅 토큰 위 드래그는 토큰 이동이 아니라 맵 팬으로 처리된다.

최종 완료 기준
-------------
- A~E 항목 중 실패 항목이 없으면 현재 안정화 로드맵은 완료로 본다.
- 실패 항목이 있으면 해당 영역 문서로 돌아가 원인을 좁힌 뒤 최소 파일만 수정한다.

ITC — FINAL INTEGRATED REGRESSION CHECKLIST
==========================================

목적
----
최근 반영된 권한 안정화, UI 정리, 이미지 처리, 장면 전환, BGM, 토큰/저널/스탠딩 퀵뷰, 토큰 공개/비공개 기능이 서로 회귀하지 않는지 최종 확인한다.
큰 기능 추가 전후에는 이 문서를 기준으로 전체 기능을 한 번 점검한다.

진행 원칙
---------
- 실패한 항목이 나오면 즉시 다음 테스트로 넘어가지 말고, 실패 영역만 좁혀서 수정한다.
- 한 번에 여러 영역을 동시에 고치지 않는다.
- Firebase Rules / Cloudinary / Vercel 설정 변경이 필요하면 코드 수정 전에 먼저 확인한다.
- GM 계정 1개, 플레이어 계정 1~2개를 동시에 켜고 확인한다.
- 플레이어 권한 차이는 반드시 권한 부여 전 / 권한 부여 후 / 권한 회수 후로 나눠 확인한다.

A. 방 / 로비 / 세션
------------------
1. 새 방 생성이 정상 작동한다.
2. 생성한 방에 GM 계정이 정상 입장한다.
3. 플레이어 계정이 초대/방 코드로 정상 입장한다.
4. 플레이어 나가기 후 재입장이 가능하다.
5. GM이 방 종료/삭제를 했을 때 방 데이터가 정상 삭제된다.
6. 로비의 내 세션 목록에서 방 삭제가 정상 작동한다.
7. 삭제된 방이 새로고침 후에도 다시 나타나지 않는다.

B. 채팅 / 잡담 / 귓말 / DM
--------------------------
1. 일반 채팅 전송이 정상 작동한다.
2. 잡담 탭 메시지 전송이 정상 작동한다.
3. speak-as 선택 후 저널 이름 / 아바타 / nameColor가 적용된다.
3-1. 다른 플레이어가 설정한 저널 프로필 사진과 nameColor가 GM 화면에서도 동일하게 보인다.
3-2. 저널 시트 저장/권한 수정 후에도 nameColor가 기본 하늘색으로 회귀하지 않는다.
4. 오너 계정끼리 귓말이 정상 작동한다.
5. 저널을 선택한 상태에서 저널 이름으로 귓말이 전송된다.
6. 귓말 대상 드롭다운에 각 플레이어 오너 계정과 현재 선택 중인 저널이 표시된다.
7. 상대가 저널 선택을 해제하면 귓말 대상 목록에서도 해당 저널이 빠진다.
8. GM이 DM 채널을 생성할 수 있다.
9. 플레이어는 자신이 포함된 DM만 볼 수 있다.
10. DM 채널 전환 시 각 채널의 메시지 내역이 유지된다.
11. 다른 채널에 새 DM이 오면 unread dot이 정상 표시된다.
12. 팝아웃에서도 DM 버튼과 unread dot이 유지된다.

C. 팝아웃
---------
1. 기본창에서 일반 채팅 전송 → 팝아웃에도 동일하게 표시된다.
2. 기본창에서 desc 전송 → 팝아웃에서도 지문 형태로 표시된다.
3. 팝아웃에서 desc 전송 → 기본창/팝아웃 모두 동일하게 표시된다.
4. 굵게/기울기/굵게기울기 문법이 기본창/팝아웃에서 동일하게 보인다.
5. 기본창 이미지 전송 → 팝아웃에서 URL 텍스트가 아니라 이미지로 표시된다.
6. speak-as 이미지 전송 → 팝아웃에서 저널 이름 / 아바타 / nameColor가 유지된다.
7. 기본창 맨 위 스크롤 → 이전 메시지가 추가 로딩된다.
8. 팝아웃 맨 위 스크롤 → 이전 메시지가 추가 로딩된다.
9. DM 채널에서도 desc / 이미지 / 이전 메시지 로딩이 유지된다.
10. 팝아웃을 닫았다 다시 열어도 상단 탭과 DM 버튼이 유지된다.

D. 이미지 전송 / 표시
--------------------
1. + 버튼으로 이미지를 선택해도 즉시 전송되지 않고 미리보기만 뜬다.
2. 최대 4장 이미지 미리보기가 정상 표시된다.
3. 미리보기 이미지 삭제가 정상 작동한다.
4. 미리보기 이미지 순서 변경이 정상 작동한다.
5. 전송 후 이미지가 채팅창에 정상 표시된다.
6. 투명 PNG/WebP가 검은 배경 없이 투명하게 표시된다.
7. 일반 JPG/JPEG 이미지 업로드가 정상 작동한다.
8. 가로폭 채우기 옵션이 정상 작동한다.
9. 가로폭 채우기 + 닉네임 안보이게 조합에서 이미지 테두리 회색선이 보이지 않는다.
10. 새로고침 후에도 이미지가 정상 로딩된다.

E. 저널 / 핸드아웃 / 문서 권한
------------------------------
1. GM이 저널을 생성/수정/삭제할 수 있다.
2. 권한 받은 플레이어가 저널을 열람/수정할 수 있다.
3. 권한 없는 플레이어에게 저널이 보이지 않는다.
4. 플레이어는 저널 삭제 버튼을 볼 수 없다.
5. 플레이어는 저널 권한 목록을 바꿀 수 없다.
6. GM이 핸드아웃을 생성/수정/삭제할 수 있다.
7. 플레이어는 허용된 핸드아웃만 열람할 수 있다.
8. 플레이어에게 핸드아웃 편집/삭제 UI가 보이지 않는다.
9. 저널 퀵뷰가 정상으로 열린다.
10. 저널 퀵뷰 목록에 저널 프로필 이미지 미리보기가 정상 표시된다.
11. 프로필 이미지가 없는 저널은 fallback 표시가 정상 표시된다.
12. 저널 퀵뷰 다이스 롤이 정상으로 채팅에 출력된다.

F. 캐릭터 토큰 / 패널 토큰 / 토큰 권한
--------------------------------------
1. 캐릭터 토큰 버튼 클릭 시 별도 생성창 없이 `?` 토큰이 즉시 생성된다.
2. 생성된 캐릭터 토큰의 소유자가 현재 생성자로 잡힌다.
3. 캐릭터 토큰 이동/편집/삭제가 정상 작동한다.
4. 캐릭터 토큰에 스탠딩 이미지를 10개 이상 넣고 저장해도 다시 열었을 때 누락되지 않는다.
5. 캐릭터 토큰 편집창 하단 상태 문구/삭제/저장 버튼 UI가 창 밖으로 깨지지 않는다.
6. 패널 토큰 생성 창이 정상으로 열린다.
7. 패널 토큰 생성 시 캐릭터 토큰이 아니라 패널 토큰으로 생성된다.
8. 패널 토큰 앞면/뒷면 이미지가 정상 저장/전환된다.
9. 패널 토큰 클릭 액션 `채팅 보내기`가 정상 작동한다.
10. 패널 토큰 클릭 액션 `매크로`가 정상 작동한다.
11. speak-as 선택 상태에서 패널 토큰 클릭 액션이 저널 기준으로 발화된다.
12. 토큰 소유/권한 UI가 정상 표시된다.
13. 플레이어가 권한 없는 토큰을 임의 편집/삭제할 수 없다.
14. 새로고침 후 토큰 위치/이미지/권한 상태가 유지된다.
15. 캐릭터 토큰 편집창 상단에서 GM에게만 비공개/전체 공개 토글이 보인다.
16. 패널 토큰 편집창 상단에서 GM에게만 비공개/전체 공개 토글이 보인다.
17. 기존 토큰은 visibility 값이 없어도 전체 공개로 취급되어 플레이어 화면에 계속 보인다.
18. 신규 캐릭터 토큰/패널 토큰은 기본 전체 공개 상태로 생성된다.

G. 맵 / 좌표 / UI 컨트롤
-----------------------
1. 맵 하단 주사위 버튼 영역이 보이지 않는다.
2. 채팅 다이스 명령어는 정상 작동한다.
3. 맵 우측 아이콘 버튼 5개가 정상 표시된다.
4. 각 아이콘 버튼 hover 툴팁이 정상 표시된다.
5. 캐릭터 토큰 / 패널 토큰 / 맵세팅 / 맵세팅 레이어 / 장면 전환 버튼이 정상 작동한다.
6. GM 전용 버튼이 플레이어에게 기존 권한 규칙대로 숨겨진다.
7. 줌인/줌아웃해도 토큰 간 거리가 변하지 않는다.
8. 창 크기를 바꿔도 토큰 위치와 거리가 유지된다.
9. 맵 컨트롤 버튼 클릭 시 맵이 같이 드래그/패닝되지 않는다.
10. 이름 색상 변경 아이콘이 팔레트 이미지로 표시되고 색상 변경이 정상 작동한다.

H. 맵세팅 ZIP / 레이어
----------------------
1. 맵세팅 ZIP 가져오기 창이 정상으로 열린다.
2. ZIP 업로드 후 검사/적용이 정상 작동한다.
3. 배경 이미지가 정상 적용된다.
4. object item이 imported panel token으로 생성된다.
5. imported panel token이 캐릭터 토큰 편집창으로 열리지 않는다.
6. 레이어 창이 정상으로 열린다.
7. 레이어 눈 아이콘으로 숨김/표시 시 실제 맵 위 imported panel token도 함께 반영된다.
8. 레이어 순서 변경 시 실제 겹침 순서도 바뀐다.
9. 단일 레이어 삭제 시 연결된 imported panel token이 함께 삭제된다.
10. 전체 삭제 시 오브젝트 레이어, 연결된 imported panel token, 배경 이미지가 함께 삭제된다.

I. 장면 전환
------------
1. 장면 전환 설정창이 정상으로 열린다.
2. 씬 카드가 가로 16:9 카드형으로 표시된다.
3. 씬 카드 드래그 순서 변경이 가능하다.
4. 창을 닫았다 열어도 씬 순서가 유지된다.
5. 새로고침 후에도 씬 순서가 유지된다.
6. 씬 우클릭 복제가 가능하다.
7. 복제 씬은 원본 바로 뒤에 생성된다.
8. 씬 더블클릭 시 LIVE 씬이 실제 맵에 반영된다.
9. 씬 A에 맵세팅 ZIP 적용 후 저장 → 씬 B 전환 → 씬 A 복귀 시 배경/오브젝트/레이어 상태가 유지된다.
10. 씬 A에서 전체 삭제 후 저장 → 씬 B 전환 → 씬 A 복귀 시 삭제 상태가 유지된다.
11. 새로고침 후 active 씬 상태가 유지된다.

J. BGM
------
1. GM은 BGM 바에서 곡 추가/재생/일시정지/이전/다음/반복/볼륨 조절이 가능하다.
2. 권한 없는 플레이어는 현재 곡 정보와 재생바를 볼 수 있고, 개인 볼륨만 조절할 수 있다.
3. BGM 관리 권한을 받은 플레이어는 GM과 동일하게 BGM을 조작할 수 있다.
4. BGM 관리 권한을 회수하면 조작 버튼/삭제/드래그/선택 재생 UI가 즉시 사라지거나 비활성화된다.
5. +BGM으로 곡을 추가하면 제목이 자동 추출되고 즉시 현재 곡으로 재생된다.
6. 현재 재생 중인 곡과 같은 곡을 다시 추가하면 토스트 후 차단된다.
7. 반복 버튼은 반복 없음 → 현재 곡 반복 → 플레이리스트 전체 반복 순서로 바뀐다.
8. 반복 상태 변경 토스트가 정상 표시된다.
9. 재생바가 현재 위치를 표시한다.
10. GM/BGM 권한자는 재생바 클릭/드래그로 seek가 가능하다.
11. 권한 없는 플레이어는 재생바를 보기만 하고 이동할 수 없다.
12. 펼치기 버튼 클릭 시 하단에서 BGM 패널이 펼쳐진다.
13. 펼친 패널에 썸네일 / 곡 제목 / 플레이리스트가 표시된다.
14. 펼친 패널에서 곡 추가/삭제/선택 재생이 정상 작동한다.
15. 플레이리스트 드래그 순서 변경이 정상 작동한다.
16. 변경된 플레이리스트 순서가 접기/펼치기/+BGM 모달에 동일하게 반영된다.
17. 새로고침 후 현재 곡 / 재생 위치 / 반복 상태 / 플레이리스트 순서가 유지된다.
18. GM 새로고침 후 재생 버튼을 다시 누르지 않아도 이어서 재생된다.
19. 일시정지 상태에서 새로고침하면 자동 재생되지 않고 일시정지 상태가 유지된다.
20. 곡 삭제 후 currentTrack이 꼬이지 않는다.

K. 스탠딩 퀵뷰
----------------
1. 저널 퀵뷰 버튼 왼쪽에 스탠딩 퀵뷰 버튼이 정상 표시된다.
2. 스탠딩 퀵뷰 버튼 클릭 시 선택 중인 저널에 연결된 토큰의 스탠딩 목록이 표시된다.
3. 스탠딩 카드는 정사각형 이미지 미리보기와 `@(스탠딩 이름)` 형태로 표시된다.
4. 현재 선택된 스탠딩 카드가 금색 테두리/강조 상태로 표시된다.
5. 스탠딩이 10개 이상일 때 퀵뷰 내부 스크롤이 정상 작동한다.
6. 마우스가 스탠딩 퀵뷰 위에 있을 때 휠 스크롤은 퀵뷰 내부 스크롤만 움직이고 맵 줌은 작동하지 않는다.
7. 마우스가 맵 영역 위에 있을 때 휠 스크롤은 기존처럼 맵 줌인/줌아웃으로 작동한다.
8. 스탠딩 카드 선택 시 맵 위 토큰 이미지가 변경된다.
9. 스탠딩 변경이 GM/플레이어 화면에 동일하게 동기화된다.
10. 이후 해당 저널로 채팅을 보내면 선택된 스탠딩 기준으로 출력된다.
11. 권한 없는 플레이어는 스탠딩 변경이 차단된다.
12. 다른 유저가 같은 토큰의 스탠딩을 바꾸면 열려 있는 퀵뷰의 선택 강조도 갱신된다.
13. 퀵뷰가 열린 상태에서 speak-as 저널을 바꾸면 목록이 새 저널 기준으로 갱신된다.
14. Esc 키로 스탠딩 퀵뷰가 닫힌다.

L. 토큰 공개 / 비공개
--------------------
1. GM이 캐릭터 토큰을 비공개로 저장하면 플레이어 화면에서 해당 토큰이 사라진다.
2. GM이 패널 토큰을 비공개로 저장하면 플레이어 화면에서 해당 토큰이 사라진다.
3. GM 화면에서는 비공개/전체 공개 토큰이 모두 계속 보인다.
4. 비공개 캐릭터 토큰은 플레이어 상태 패널에도 표시되지 않는다.
5. 플레이어가 토큰 편집창을 열어둔 상태에서 GM이 해당 토큰을 비공개로 바꾸면 편집창이 자동으로 닫힌다.
6. 비공개 패널 토큰의 클릭 액션이 플레이어 화면에서 실행되지 않는다.
7. 플레이어는 비공개 토큰을 이동/편집/삭제할 수 없다.
8. 플레이어는 비공개 토큰의 스탠딩을 스탠딩 퀵뷰로 변경할 수 없다.
9. GM이 다시 전체 공개로 저장하면 플레이어 화면에 토큰이 재표시된다.
10. 장면 전환 / 씬 복제 / 씬 순서 변경 후에도 토큰 visibility 상태가 유지된다.
11. 공개 토큰의 기존 이동/편집/스탠딩 퀵뷰/패널 토큰 클릭 기능은 그대로 유지된다.

M. 최종 완료 기준
-----------------
- A~L 항목 중 실패 항목이 없으면 현재 안정화 로드맵은 완료로 본다.
- 실패 항목이 있으면 해당 영역만 최소 범위로 수정한다.
- 전체 통합 테스트 통과 후 Git 커밋을 권장한다.

Firebase Rules Stage 4 콘솔 적용 테스트 체크리스트
====================================================

현재 상태
---------
- Firebase Console에 `database.rules.console-candidate.json` 적용 완료.
- GM/플레이어 2계정 테스트 통과.
- `database.rules.json`은 현재 운영 적용 Rules 기준으로 승격됨.
- `database.rules.rollback-current.json`은 하드닝 전 기존 Rules 롤백본으로 유지.

적용 후 통과 확인
----------------
[x] GM 계정으로 새 방 생성 가능
[x] 일반 플레이어가 방 코드로 입장 가능
[x] 일반 플레이어가 방 나가기 가능
[x] 일반 플레이어가 마지막으로 나가도 방 자체가 삭제되지 않음
[x] GM/owner는 방 종료 또는 로비 세션 삭제 가능
[x] 채팅/잡담/desc/이미지/팝아웃 주요 기능 정상
[x] GM 토큰 생성/이동/편집/삭제 정상
[x] GM 맵세팅 ZIP/레이어/전체 삭제 정상
[x] 장면 전환 저장/복제/카드 순서 변경 정상
[x] 저널/핸드아웃 기존 기능 회귀 없음
[보류] BGM manageBgm 실제 기능 테스트

향후 권한 UI 구현 후 재확인할 항목
-------------------------------
[ ] GM이 플레이어 permissions 부여/회수 가능
[ ] createToken 권한 플레이어 토큰 생성 가능
[ ] moveToken 권한 플레이어 토큰 이동 가능
[ ] editToken 권한 플레이어 토큰 편집/삭제 가능
[ ] 권한 회수 후 해당 기능 차단
[ ] manageMap 권한 플레이어 맵 관련 기능 범위 확인
[ ] manageBgm 권한 플레이어 BGM 기능 확인

문제 발생 시 즉시 롤백
--------------------
아래 중 하나라도 발생하면 운영 테스트를 중단하고 즉시 롤백한다.

- 방 입장 불가
- 채팅 전송 불가
- GM 방 삭제 불가
- GM 맵세팅 ZIP 업로드 불가
- GM 토큰 생성/편집 불가
- mapScenes 저장 실패
- 예상하지 않은 PERMISSION_DENIED 반복

롤백 방법
--------
1. `database.rules.rollback-current.json` 내용을 연다.
2. Firebase Console > Realtime Database > Rules에 전체 붙여넣기 한다.
3. Publish 한다.
4. GM/플레이어 양쪽 브라우저를 새로고침한다.
5. 방 입장, 채팅, GM 토큰 생성, 맵세팅 ZIP 업로드, 장면 전환 저장을 다시 확인한다.

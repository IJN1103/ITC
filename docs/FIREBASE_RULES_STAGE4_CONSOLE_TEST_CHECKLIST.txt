Firebase Rules Stage 4 콘솔 적용 테스트 체크리스트
====================================================

목적
----
이 문서는 `database.rules.console-candidate.json`을 Firebase Console에 실제로 적용한 뒤,
운영 기능이 막히지 않는지 확인하기 위한 테스트 순서다.

중요
----
- 이 단계부터는 Firebase Console에서 직접 Rules를 바꾸는 외부 작업이 필요하다.
- 적용 전 반드시 현재 콘솔 Rules를 백업한다.
- 프로젝트 안에는 롤백용으로 `database.rules.rollback-current.json`을 함께 둔다.
- 문제가 생기면 즉시 `database.rules.rollback-current.json` 내용을 Firebase Console Rules에 다시 붙여넣어 롤백한다.

적용 전 준비
-----------
[ ] Firebase Console > Realtime Database > Rules 화면 열기
[ ] 현재 콘솔 Rules 전체를 별도 메모장/파일에 복사해 백업
[ ] 프로젝트 파일의 `database.rules.rollback-current.json` 내용이 현재 콘솔 Rules와 같은지 확인
[ ] `database.rules.console-candidate.json` 내용을 열어 복사 준비
[ ] 테스트용 방 1개 준비
[ ] GM 계정 1개, 일반 플레이어 계정 1개 준비

Console 적용 순서
----------------
1. Firebase Console > Realtime Database > Rules로 이동한다.
2. 현재 Rules 전체를 다시 한 번 백업한다.
3. `database.rules.console-candidate.json` 전체 내용을 붙여넣는다.
4. Publish 한다.
5. 바로 아래 테스트를 진행한다.

필수 테스트 1: 방 생성 / 입장 / 퇴장
-----------------------------------
[ ] GM 계정으로 새 방 생성 가능
[ ] 일반 플레이어가 방 코드로 입장 가능
[ ] 일반 플레이어가 방 나가기 가능
[ ] 일반 플레이어가 마지막으로 나가도 방 자체가 삭제되지 않음
[ ] GM/owner는 방 종료 또는 로비 세션 삭제 가능
[ ] 일반 플레이어가 로비에서 세션 X를 눌러도 방 전체 삭제가 되지 않음

필수 테스트 2: players 권한 상승 방지
-----------------------------------
[ ] 일반 플레이어가 자기 role을 gm으로 바꿀 수 없음
[ ] 일반 플레이어가 자기 permissions를 직접 추가/수정할 수 없음
[ ] GM/owner는 플레이어 권한 관리가 가능
[ ] 일반 플레이어의 닉네임/아바타/색상/접속 표시 등 기본 정보가 기존처럼 유지됨

필수 테스트 3: 채팅 / DM / 팝아웃
--------------------------------
[ ] 일반 채팅 전송 가능
[ ] 잡담 전송 가능
[ ] desc 지문 전송 가능
[ ] 팝아웃 desc 스타일 정상
[ ] 팝아웃 이미지가 URL 텍스트로 뜨지 않음
[ ] 기본창/팝아웃 이전 채팅 로딩 정상
[ ] DM 채널 생성은 GM만 가능
[ ] DM 메시지 송수신 정상
[ ] 팝아웃 DM 버튼/unread dot 정상

필수 테스트 4: 토큰 권한
-----------------------
[ ] GM은 캐릭터 토큰 생성/이동/편집/삭제 가능
[ ] GM은 패널 토큰 생성/이동/편집/삭제 가능
[ ] 권한 없는 플레이어는 토큰 생성/편집/삭제가 차단됨
[ ] moveToken 권한이 있는 플레이어는 토큰 이동 가능
[ ] createToken 권한이 있는 플레이어는 토큰 생성 가능
[ ] editToken 권한이 있는 플레이어는 토큰 편집/삭제 가능
[ ] 권한 회수 후 해당 기능이 다시 차단됨

필수 테스트 5: 맵세팅 / 레이어 / 장면 전환
-----------------------------------------
[ ] GM은 맵세팅 ZIP 업로드 가능
[ ] 맵세팅 오브젝트가 imported panel token으로 생성됨
[ ] 위치고정 imported panel token 위 드래그 시 토스트 없이 맵 팬
[ ] 맵세팅 레이어 숨김/표시 정상
[ ] 맵세팅 레이어 순서 변경 정상
[ ] 단일 삭제 정상
[ ] 전체 삭제 정상
[ ] 배경만 남은 상태에서도 전체 삭제 가능
[ ] 씬 전환 정상
[ ] 씬 저장 정상
[ ] 씬 복제 정상
[ ] 씬 카드 드래그 순서 변경/새로고침 유지 정상
[ ] 맵세팅이 적용된 씬을 저장 후 다른 씬으로 갔다 돌아와도 상태 유지

필수 테스트 6: 저널 / 핸드아웃
------------------------------
[ ] GM은 저널 생성/편집 가능
[ ] 권한 받은 플레이어는 저널 보기/수정 가능
[ ] 권한 없는 저널 노출 상태가 기존 기획과 같음
[ ] GM은 핸드아웃 생성/편집/삭제 가능
[ ] 플레이어는 허용된 핸드아웃만 확인 가능
[ ] 핸드아웃 편집은 플레이어에게 열리지 않음

BGM 테스트
---------
현재 BGM 기능은 완성 상태가 아니므로, 아래 항목은 기능 구축 후 별도 테스트한다.
[보류] GM은 BGM 추가/삭제/재생 가능
[보류] manageBgm 권한 플레이어는 BGM 수정 가능
[보류] 권한 없는 플레이어는 BGM 수정 불가
[보류] 권한 없는 플레이어도 GM이 바꾼 재생 상태는 수신 가능

문제 발생 시 즉시 롤백
--------------------
아래 중 하나라도 발생하면 운영 테스트를 중단하고 즉시 롤백한다.

- 방 입장 불가
- 채팅 전송 불가
- GM 방 삭제 불가
- GM 맵세팅 ZIP 업로드 불가
- GM 토큰 생성/편집 불가
- mapScenes 저장 실패
- 예상하지 않은 PERMISSION_DENIED 반복

롤백 방법
--------
1. `database.rules.rollback-current.json` 내용을 연다.
2. Firebase Console > Realtime Database > Rules에 전체 붙여넣기 한다.
3. Publish 한다.
4. 새로고침 후 기본 기능을 다시 확인한다.

통과 기준
--------
- GM/owner 기능이 막히지 않는다.
- 일반 플레이어의 권한 상승/방 삭제/무단 토큰 수정이 막힌다.
- 채팅, 팝아웃, 패널 토큰, 맵세팅, 장면 전환 기능이 기존처럼 유지된다.

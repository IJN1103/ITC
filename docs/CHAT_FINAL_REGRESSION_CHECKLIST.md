# CHAT 최종 회귀 테스트 체크리스트 — PHASE 10

기준 파일: `ITC-main(301).zip`

목적: CHAT 구조 정리/최적화 이후, 이후 기능 추가나 수정 전에 반복해서 확인할 수 있는 최종 회귀 테스트 기준을 정리한다.

이번 문서는 코드 변경 없이 체크리스트만 추가한다. 콘솔 입력이 필요한 테스트는 포함하지 않는다.

- PHASE 6-1 ~ 6-5: 팝아웃 채팅 구조 안정화
- PHASE 7: chat build 체계 점검
- PHASE 8: 채팅 성능 최적화 2차

## 1. 기본 전체 채팅

### 1-1. 일반 메시지 전송

- [ ] 전체 채팅 탭에서 일반 텍스트 메시지를 전송할 수 있다.
- [ ] 보낸 메시지가 본창에 즉시 표시된다.
- [ ] 다른 접속자에게도 같은 메시지가 표시된다.
- [ ] 같은 메시지를 연속으로 보내도 누락되지 않는다.
- [ ] 한글 문장 전송 후 마지막 글자만 별도 메시지로 전송되지 않는다.
- [ ] PC Enter 전송이 정상 동작한다.
- [ ] Shift+Enter 줄바꿈이 정상 동작한다.
- [ ] 전송 버튼 클릭 전송이 정상 동작한다.

### 1-2. 긴 메시지 / 줄바꿈

- [ ] 여러 줄 메시지가 줄바꿈을 유지해 표시된다.
- [ ] 긴 메시지가 채팅 레이아웃을 깨지 않는다.
- [ ] desc 말풍선과 일반 채팅 말풍선이 구분되어 표시된다.

## 2. 이전 채팅 히스토리 / 스크롤

### 2-1. 이전 채팅 로딩

- [ ] 채팅이 많이 쌓인 방에서 상단으로 스크롤하면 이전 채팅이 추가로 로딩된다.
- [ ] 이전 채팅 일부 구간이 비어 보이지 않는다.
- [ ] 이전 채팅 순서가 시간순으로 자연스럽게 이어진다.
- [ ] DM 메시지가 많이 섞인 방에서도 전체 채팅 이전 기록이 과도하게 누락되지 않는다.

### 2-2. 스크롤 위치 보존

- [ ] 이전 채팅을 보고 있는 중 새 메시지가 와도 현재 보고 있던 위치가 크게 튀지 않는다.
- [ ] 상단 히스토리 로딩 후 화면이 갑자기 최신 메시지로 이동하지 않는다.
- [ ] 최신 하단에 있을 때는 새 메시지 수신 후 자연스럽게 최신 위치를 유지한다.
- [ ] 메시지 DOM trim 이후에도 현재 보고 있던 영역이 비정상적으로 사라지지 않는다.

---

## 3. DM 채팅

### 3-1. DM방 진입/전송

- [ ] GM이 생성한 DM방에 참여자가 정상 진입할 수 있다.
- [ ] 플레이어는 GM이 생성하지 않은 DM방에 진입할 수 없다.
- [ ] 플레이어끼리만 DM하려 할 경우 차단 메시지가 표시된다.
- [ ] DM방에서 메시지를 전송할 수 있다.
- [ ] DM 메시지는 해당 DM방 참여자에게만 보인다.
- [ ] 전체 채팅에는 DM 메시지가 노출되지 않는다.

### 3-2. DM unread DOT

- [ ] DM방을 클릭만 해도 unread DOT가 생기지 않는다.
- [ ] DM 목록을 열고 닫기만 해도 unread DOT가 생기지 않는다.
- [ ] 본인이 DM을 보냈을 때 본인에게 unread DOT가 생기지 않는다.
- [ ] 현재 보고 있는 DM방에 본인이 메시지를 보내도 본인 DOT가 생기지 않는다.
- [ ] 새 DM을 받은 상대에게만 unread DOT가 생긴다.
- [ ] 수신자가 해당 DM방에 들어가면 unread DOT가 사라진다.
- [ ] 본창 또는 팝아웃에서 해당 DM방을 보고 있으면 read 상태로 처리된다.

### 3-3. DM방 목록/삭제

- [ ] DM방 목록 버튼이 열린다.
- [ ] GM은 전체 DM방 목록을 볼 수 있다.
- [ ] 플레이어는 본인이 포함된 DM방만 볼 수 있다.
- [ ] GM이 DM방 삭제 시 해당 방과 메시지 기록이 함께 삭제된다.
- [ ] 삭제된 DM방은 목록과 unread 상태에서 제거된다.

---

## 4. 팝아웃 채팅

### 4-1. 본창/팝아웃 채널 독립성

- [ ] 본창 전체 채팅 / 팝아웃 DM 조합이 가능하다.
- [ ] 본창 DM / 팝아웃 전체 채팅 조합이 가능하다.
- [ ] 본창 A DM / 팝아웃 B DM 조합이 가능하다.
- [ ] 팝아웃에서 채널을 바꿔도 본창 채널이 강제로 바뀌지 않는다.
- [ ] 본창에서 채널을 바꿔도 팝아웃 채널이 강제로 바뀌지 않는다.

### 4-2. 팝아웃 전체 채팅

- [ ] 팝아웃 전체 채팅에서 메시지를 전송하면 팝아웃에 즉시 표시된다.
- [ ] 팝아웃 전체 채팅에서 보낸 메시지가 본창 전체 채팅에도 표시된다.
- [ ] 본창 전체 채팅에서 보낸 메시지가 팝아웃 전체 채팅에도 표시된다.
- [ ] 팝아웃에서 연속 전송해도 누락되지 않는다.

### 4-3. 팝아웃 DM

- [ ] 팝아웃 DM방에서 기존 메시지가 보인다.
- [ ] 팝아웃 DM방에서 새 메시지를 전송하면 팝아웃에 즉시 표시된다.
- [ ] 팝아웃 DM방에서 보낸 메시지가 본창 해당 DM방에도 표시된다.
- [ ] 본창 해당 DM방에서 보낸 메시지가 팝아웃 DM방에도 표시된다.
- [ ] 팝아웃에서 DM을 보내도 본창의 기존 채널 선택은 유지된다.
- [ ] 팝아웃 DM 전송 시 본인 unread DOT가 생기지 않는다.

### 4-4. 팝아웃 기타 탭

- [ ] 팝아웃 잡담탭이 정상 표시된다.
- [ ] 팝아웃 문서/저널 관련 표시가 기존처럼 유지된다.
- [ ] 팝아웃 이름 색상/저널 선택/speak-as 표시가 기존처럼 유지된다.
- [ ] 팝아웃을 닫은 뒤 본창 채팅/DM이 정상 동작한다.

## 5. 메시지 수정/삭제

### 5-1. PC hover 액션

- [ ] PC에서 본인 메시지에 마우스를 올리면 수정/삭제 버튼이 보인다.
- [ ] PC에서 다른 유저 메시지는 권한 없는 수정/삭제 버튼이 보이지 않는다.
- [ ] GM은 다른 유저 메시지를 삭제할 수 있다.
- [ ] 플레이어는 다른 유저 메시지를 삭제할 수 없다.
- [ ] 수정 후 본창 메시지가 갱신된다.
- [ ] 삭제 후 본창 메시지가 제거된다.
- [ ] 수정/삭제가 팝아웃에도 반영된다.

### 5-2. 모바일 long-press 액션

- [ ] 모바일에서는 수정/삭제 버튼이 평상시 보이지 않는다.
- [ ] 본인 메시지를 길게 누르면 수정/삭제 옵션 창이 뜬다.
- [ ] 모바일에서 수정이 정상 동작한다.
- [ ] 모바일에서 삭제가 정상 동작한다.
- [ ] 채팅창 스크롤 중 실수로 옵션 창이 자주 뜨지 않는다.
- [ ] 이미지/주사위 등 기존 수정 불가 메시지는 수정 옵션이 뜨지 않는다.

## 6. 이미지 채팅

- [ ] 이미지 선택 버튼으로 이미지를 추가할 수 있다.
- [ ] 이미지 프리뷰가 표시된다.
- [ ] 여러 이미지 선택 시 순서 변경이 가능하다.
- [ ] 프리뷰 X 버튼으로 이미지 제거가 가능하다.
- [ ] 이미지 전송 후 본창에 표시된다.
- [ ] 이미지 전송 후 팝아웃에도 표시된다.
- [ ] 이미지가 과도하게 늘어나거나 찌그러지지 않는다.
- [ ] 이미지 lazy loading/placeholder가 레이아웃을 크게 흔들지 않는다.
- [ ] 전체 채팅과 DM 모두 이미지 전송이 정상 동작한다.

## 7. 잡담탭 / 귓말 / 다이스

### 7-1. 잡담탭

- [ ] 잡담탭에서 메시지를 전송할 수 있다.
- [ ] 잡담 닉네임과 색상이 유지된다.
- [ ] 잡담 메시지 수정/삭제가 기존 권한대로 동작한다.
- [ ] 잡담 이전 메시지 로딩이 유지된다.

### 7-2. 귓말

- [ ] 귓말 대상 목록이 열린다.
- [ ] 귓말을 전송할 수 있다.
- [ ] 발신자와 수신자에게만 귓말이 보인다.
- [ ] 권한 없는 유저에게 귓말 내용이 보이지 않는다.
- [ ] 팝아웃에서도 귓말 표시가 기존처럼 유지된다.

### 7-3. 다이스

- [ ] `/1d100` 등 다이스 명령이 정상 전송된다.
- [ ] 다이스 결과가 본창에 표시된다.
- [ ] 다이스 결과가 팝아웃에도 표시된다.
- [ ] 다이스 메시지는 기존 수정 제한을 유지한다.

## 8. 저널 speak-as / 스탠딩

- [ ] 저널 선택 드롭다운이 열린다.
- [ ] 선택한 저널로 채팅을 보낼 수 있다.
- [ ] 선택한 저널의 이름/색상/프로필이 반영된다.
- [ ] `@스탠딩명` 명령으로 스탠딩 변경이 적용된다.
- [ ] 두상 노출 ON/OFF 상태가 말풍선 표시와 맞게 반영된다.
- [ ] 팝아웃에서도 저널 speak-as 표시가 유지된다.
- [ ] 모바일에서 저널 선택 드롭다운이 잘리지 않고 선택된다.

- [ ] 팝아웃 입력창 테두리 드래그로 높이 조절이 가능하다.

- [ ] 모바일에서는 의도치 않은 입력창 resize가 발생하지 않는다.

## 14. 현재 기준 남은 장기 검토

PHASE 9로 남겨둘 장기 검토 항목:

- 전체 채팅과 DM 메시지를 같은 `rooms/{roomCode}/chat` 경로에 저장하는 구조의 장단점 검토
- `rooms/{roomCode}/chat/public` / `rooms/{roomCode}/dmChats/{dmKey}/messages` 분리 가능성 검토
- Firebase Rules 수정 범위 검토
- 기존 데이터 마이그레이션 필요 여부 검토
- 팝아웃 watcher/unread/DM 삭제 로직 영향 검토

이 항목은 위험도가 높기 때문에, 바로 구현하지 않고 먼저 검토 문서로 분리하는 것이 안전하다.

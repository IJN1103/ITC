ITC-main — NEXT SESSION HANDOFF (2025-06 갱신)
================================================

1) 프로젝트 개요
----------------
- TRPG 채팅 기반 웹앱
- 한 방 최대 5명, 사이트 전체 목표 300명
- 채팅량 많음, 이미지 업로드 빈번, 방 장시간 유지
- "바이브 코딩" 방식 — 파일 덮어쓰기 중심, 부분 코드 금지

2) 작업 방식 규칙
-----------------
- 변경된 파일만 전달
- 파일명 절대 변경 금지
- 항상 전체 파일 제공 (부분 코드 금지)
- 구조 리팩토링은 지시 없이 금지
- 한 번에 하나의 문제만, 최소 수정, 안정성 최우선

3) 핵심 아키텍처
----------------
[좌표계]
- 토큰 좌표: % 기반 단일 좌표계
- stored = display (변환 없음)
- 드래그 계산: % 기준
- zoom/scale: CSS transform에서만 사용
- map-inner: 고정 논리 캔버스 (최초 유효 크기로 잠금, 이후 불변)
- map-area: 뷰포트

[절대 깨면 안 되는 규칙]
- 토큰 좌표 계산에 scale 넣지 말 것
- expansion 같은 보정 다시 만들지 말 것
- stored/display 변환 분리하지 말 것
- 드래그 계산에 px 기준 쓰지 말 것

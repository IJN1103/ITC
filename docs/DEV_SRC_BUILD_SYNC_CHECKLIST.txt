DEV / SRC 빌드 동기화 체크리스트
=================================

목적
----
최근 채팅/팝아웃, 패널 토큰, 맵세팅, 장면 전환 안정화 작업은 src 기준으로 많이 진행되었다.
이 상태에서 build.sh가 오래된 dev 파일을 다시 합치면 최근 수정이 대량 회귀될 수 있다.
따라서 2026-04-20 기준으로 dev 파일을 최신 src 내용에 맞춰 다시 동기화했다.

동기화 대상
----------
아래 세 묶음은 build.sh 실행 시 src 파일을 생성한다.

1. 채팅
- dev/chat-v1/_chat-render.js
- dev/chat-v1/_chat-image.js
- dev/chat-v1/_chat-send.js
- src/modules/chat-v1/chat.js

2. 맵/토큰
- dev/map-token/_map-viewport.js
- dev/map-token/_map-render.js
- dev/map-token/_token-edit.js
- src/modules/map-token/map-token.js

3. 저널
- dev/journal-v1/_journal-base.js
- dev/journal-v1/_journal-core.js
- dev/journal-v1/_journal-sheet.js
- src/modules/journal-v1/journal.js

검증 방법
----------
프로젝트 루트에서 아래 명령을 실행한다.

```bash
bash build.sh
```

정상 기준:
- chat.js 2145 lines
- map-token.js 1989 lines
- journal.js 2402 lines

주의사항
--------
- build.sh 실행 후 채팅/팝아웃 desc, 이미지, 이전 채팅 로딩이 회귀되면 안 된다.
- build.sh 실행 후 패널 토큰, 맵세팅 imported panel token, 장면 전환 연동이 회귀되면 안 된다.
- src만 직접 수정한 경우 반드시 dev에도 같은 내용을 반영한다.
- dev만 수정한 경우 반드시 build.sh를 실행해서 src도 갱신한다.
- 다음 AI가 dev 파일만 보고 작업하더라도 최신 기능 기준을 잃지 않도록 이 문서를 유지한다.

최소 회귀 테스트
---------------
1. 팝아웃 desc가 기본창과 같은 흰색 볼드 지문인지 확인
2. 팝아웃 이미지가 URL 텍스트가 아니라 이미지로 뜨는지 확인
3. 기본창/팝아웃 상단 스크롤 시 이전 메시지가 보이는지 확인
4. +패널 토큰 생성 시 캐릭터 토큰이 아니라 패널 토큰으로 생성되는지 확인
5. 맵세팅 ZIP object가 imported panel token으로 생성되는지 확인
6. 위치고정 imported panel token 위 좌클릭 드래그 시 토스트 없이 맵 팬이 되는지 확인
7. 씬 전환 후 맵세팅 레이어 숨김/표시/순서가 유지되는지 확인

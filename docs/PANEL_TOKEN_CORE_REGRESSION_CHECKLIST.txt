패널 토큰 코어 회귀 테스트 체크리스트

목적
- 패널 토큰 관련 수정 이후, 기존 핵심 기능이 깨지지 않았는지 같은 순서로 확인하기 위한 공통 테스트 문서입니다.
- 이 문서는 패널 토큰 코어 안정화 1~6단계를 지난 기준선 테스트 묶음입니다.
- 앞으로 패널 토큰, 맵 토큰, speak-as, 이미지, 채팅, 렌더링 관련 수정 후에는 가능하면 이 순서대로 점검합니다.

기본 원칙
- 한 번에 여러 기능을 섞어서 보지 말고, 아래 순서를 그대로 따릅니다.
- 각 테스트는 가능하면 “성공 / 실패 / 애매함”으로 기록합니다.
- 실패가 나오면 그 시점에서 중단하고, 어느 단계에서 실패했는지 바로 남깁니다.

사전 준비
1. 테스트용 방에 입장
2. 패널 토큰 생성 권한 / 편집 권한 확인
3. speak-as용 저널 1개 이상 준비
4. 앞면/뒷면 테스트용 이미지 2장 준비
5. 일반 채팅과 다이스 채팅이 보이는 상태 확인

━━━━━━━━━━━━━━━━━━
A. 생성 / 기본 저장
━━━━━━━━━━━━━━━━━━
1. 패널 토큰 생성
2. 이름 입력 후 생성 확인
3. 편집창 열기
4. 아래 값 저장
   - 이름
   - 폭
   - 높이
   - 겹침 우선도
   - 메모
5. 저장 후 다시 열기
6. 방금 저장한 값이 그대로 유지되는지 확인

성공 기준
- 저장값이 누락되지 않음
- 다시 열었을 때 이전 값이 정확히 보임

━━━━━━━━━━━━━━━━━━
B. 이미지 상태
━━━━━━━━━━━━━━━━━━
1. 앞면 이미지만 추가 후 저장
2. 다시 열어 앞면만 유지되는지 확인
3. 뒷면 이미지만 추가 후 저장
4. 다시 열어 뒷면만 유지되는지 확인
5. 앞면 + 뒷면 둘 다 추가 후 저장
6. 다시 열어 둘 다 유지되는지 확인
7. 앞면만 삭제 후 저장
8. 뒷면은 유지되고 앞면만 사라지는지 확인
9. 뒷면만 삭제 후 저장
10. 앞면은 유지되고 뒷면만 사라지는지 확인

성공 기준
- 앞면/뒷면 상태가 서로 섞이지 않음
- 한쪽 삭제가 다른 쪽에 영향 주지 않음

━━━━━━━━━━━━━━━━━━
C. 클릭 액션 저장
━━━━━━━━━━━━━━━━━━
1. 클릭 액션 = 사용안함
2. 저장 후 클릭 시 아무 동작 없는지 확인
3. 클릭 액션 = 채팅 보내기
4. 일반 텍스트 입력 후 저장
5. 클릭 액션 = 매크로
6. /1d10 또는 /choice(a,b) 저장
7. 채팅 보내기에 /로 시작하는 문장 입력 시 저장 차단되는지 확인
8. 매크로에 / 없이 입력 시 저장 차단되는지 확인
9. 빈값 저장 차단되는지 확인

성공 기준
- 검증 규칙이 정확히 동작함
- 잘못된 입력이 저장되지 않음

━━━━━━━━━━━━━━━━━━
D. 클릭 충돌
━━━━━━━━━━━━━━━━━━
1. 채팅 보내기 패널 토큰 준비
2. 단일 클릭 시 1회만 발화하는지 확인
3. 빠르게 연속 클릭 시 비정상 중복 발화 없는지 확인
4. 앞/뒷면 이미지 둘 다 있는 상태에서 더블클릭
5. 더블클릭 시 전환만 되고 클릭 액션은 같이 실행되지 않는지 확인
6. 우클릭 시 컨텍스트 메뉴만 열리는지 확인
7. 우클릭 직후 클릭 액션이 늦게 실행되지 않는지 확인
8. 드래그 이동 후 클릭 액션이 오발동하지 않는지 확인
9. 다중 선택 이동 후에도 오발동 없는지 확인
10. erase 툴에서 삭제 시 클릭 액션이 실행되지 않는지 확인

성공 기준
- 단일 클릭 / 더블클릭 / 우클릭 / 드래그 / 삭제가 서로 충돌하지 않음

━━━━━━━━━━━━━━━━━━
E. speak-as 연동
━━━━━━━━━━━━━━━━━━
1. speak-as 미선택 상태에서 채팅 보내기 실행
2. 기본 계정으로 발화되는지 확인
3. speak-as 저널 선택
4. 같은 패널 토큰 다시 실행
5. 저널 이름 / 프로필 / 스탠딩 / 대사창 기준이 따라가는지 확인
6. /choice(...) 실행
7. 일반 채팅 보내기와 같은 기준으로 발화되는지 확인
8. 일반 다이스 매크로 실행
9. 이름 / 프로필 / 스탠딩 관련 표시가 같은 기준으로 나가는지 확인
10. 다른 저널로 speak-as 변경 후 다시 실행
11. 현재 선택 저널 기준으로 바뀌는지 확인
12. speak-as 해제 후 기본 계정 fallback 확인

성공 기준
- 채팅 / choice / 다이스가 모두 같은 speak-as 기준을 따름

━━━━━━━━━━━━━━━━━━
F. 편집창 생명주기
━━━━━━━━━━━━━━━━━━
1. 패널 토큰 A 편집창 열기
2. 수정하지 않고 닫기
3. 패널 토큰 B 편집창 열기
4. A의 값이 B에 남지 않는지 확인
5. A 편집창 열기
6. 이름/메모/액션 텍스트 등을 수정하되 저장하지 않기
7. 외부 갱신 또는 렌더 갱신 유도
8. 로컬 draft가 덮어써지지 않는지 확인
9. 수정 없는 상태에서 외부 갱신 또는 렌더 갱신 유도
10. 최신 토큰 상태가 모달에 반영되는지 확인
11. 편집 중이던 패널 토큰 삭제
12. 편집창이 자동으로 닫히는지 확인

성공 기준
- 이전 토큰 draft가 다음 토큰에 남지 않음
- dirty 상태는 보호되고, clean 상태만 자동 동기화됨
- 삭제된 토큰 편집창은 자동 정리됨

━━━━━━━━━━━━━━━━━━
G. 렌더 / 재렌더
━━━━━━━━━━━━━━━━━━
1. 패널 토큰에 채팅 보내기 또는 매크로 설정
2. 클릭 정상 실행 확인
3. 저장 직후 다시 클릭
4. 저장 전 상태의 늦은 액션이 튀어나오지 않는지 확인
5. 앞/뒷면 둘 다 있는 패널 토큰 더블클릭 전환
6. 전환 후 늦은 클릭 액션이 남지 않는지 확인
7. 전체 렌더가 다시 도는 상황 확인
8. 이전 렌더 기준의 늦은 발화가 남지 않는지 확인
9. 개별 토큰만 갱신되는 상황 확인
10. 개별 재렌더 후에도 클릭 상태가 꼬이지 않는지 확인

성공 기준
- 재렌더 이후 이전 클릭 예약/락 상태가 잔존하지 않음

━━━━━━━━━━━━━━━━━━
H. 최종 마감 체크
━━━━━━━━━━━━━━━━━━
아래가 모두 만족되면 패널 토큰 코어 기준선은 통과로 봅니다.
- 저장/삭제/전환 흐름 안정
- 앞면/뒷면 이미지 상태 안정
- 클릭 충돌 없음
- speak-as 연동 안정
- 편집창 생명주기 안정
- 재렌더 후 늦은 발화 없음

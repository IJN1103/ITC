# 저널 / 핸드아웃 권한 체크리스트

## 현재 운영 기준

### 저널
- GM은 모든 저널을 열람/수정/삭제할 수 있다.
- 플레이어는 본인이 만든 저널 또는 GM이 권한을 준 저널만 열람/수정할 수 있다.
- 저널 삭제는 GM만 가능하다.
- 플레이어에게는 저널 목록 / 기본 저널 편집창 / 캐릭터 시트 창의 삭제 버튼이 보이면 안 된다.
- `assignedTo` 배열은 기존 UI 표시/호환용으로 유지한다.
- `assignedMap` 객체는 Firebase Rules에서 권한 판정하기 위한 보조 필드다.
- GM이 저널 권한을 저장하면 `assignedTo`와 `assignedMap`이 함께 저장되어야 한다.

### 핸드아웃
- GM은 모든 핸드아웃을 생성/열람/수정/삭제할 수 있다.
- 플레이어는 GM이 열람 권한을 준 핸드아웃만 볼 수 있다.
- 플레이어는 핸드아웃을 수정/삭제할 수 없다.
- `allowedTo` 배열은 기존 UI 표시/호환용으로 유지한다.
- `allowedMap` 객체는 이후 per-item read 하드닝을 위한 보조 필드다.

## 테스트 순서

1. GM으로 새 저널을 만든다.
2. GM이 특정 플레이어에게 저널 권한을 부여하고 저장한다.
3. 플레이어 계정에서 해당 저널이 보이는지 확인한다.
4. 플레이어 계정에서 해당 저널 내용을 수정/저장할 수 있는지 확인한다.
5. 플레이어 계정에서 해당 저널 목록의 삭제 버튼이 보이지 않는지 확인한다.
6. 플레이어 계정에서 해당 저널을 열었을 때 기본 저널 편집창의 삭제 버튼이 보이지 않는지 확인한다.
7. 플레이어 계정에서 해당 저널의 캐릭터 시트 창을 열었을 때 삭제 버튼이 보이지 않는지 확인한다.
8. GM 계정에서 해당 저널 삭제가 가능한지 확인한다.
9. GM으로 핸드아웃을 만든다.
10. 플레이어에게 핸드아웃 열람 권한을 부여하고 저장한다.
11. 플레이어 계정에서 해당 핸드아웃을 열람할 수 있는지 확인한다.
12. 플레이어 계정에서 핸드아웃 편집 툴바/저장/삭제 버튼이 보이지 않는지 확인한다.
13. GM 계정에서 핸드아웃 수정/삭제가 가능한지 확인한다.

## 주의사항

- Firebase Rules를 변경한 뒤에는 반드시 GM/플레이어 2계정으로 테스트한다.
- 저널 권한을 받은 플레이어가 권한 목록 자체를 바꾸면 안 된다.
- 핸드아웃은 현재 전체 read 후 프론트 필터링 구조다. per-item read 하드닝은 로딩 구조를 바꾼 뒤 별도 작업으로 진행한다.

Firebase Rules 하드닝 체크리스트
================================

현재 상태
---------
- 2026-04-20 기준 Stage 4 Console 적용 및 GM/플레이어 2계정 테스트 통과.
- `database.rules.json`은 현재 운영 적용 Rules 기준이다.
- `database.rules.rollback-current.json`은 하드닝 적용 전 롤백용 기존 Rules다.
- BGM 기능은 아직 구축되지 않아 manageBgm 실제 기능 테스트는 보류한다.

1. 적용 후 핵심 확인
-------------------
[x] `database.rules.console-candidate.json` Firebase Console 적용 완료
[x] `database.rules.json`을 운영 적용 Rules 기준으로 승격
[x] `database.rules.rollback-current.json` 롤백본 유지
[x] GM/플레이어 2계정 기본 테스트 통과
[x] 방 삭제는 GM/owner만 가능
[x] 일반 플레이어가 마지막으로 나가도 방 전체 삭제 안 됨

2. 방 / players 권한
------------------
[x] GM/owner는 방 종료 또는 로비 세션 삭제 가능
[x] 일반 플레이어는 방 전체 삭제 불가
[x] 일반 플레이어 role/permissions 직접 상승 차단
[ ] 추후 권한 UI 추가 시 GM이 role/permissions를 의도대로 부여/회수하는지 재확인

3. Chat / DM / Popout 회귀 테스트
--------------------------------
[x] 일반 채팅 전송 가능
[x] 잡담 전송 가능
[x] desc 지문 기본창/팝아웃 동일 스타일
[x] 이미지가 팝아웃에서 URL 텍스트로 보이지 않음
[x] speak-as 이미지 정상
[x] @스탠딩이름 화면 표시 숨김
[x] 기본창/팝아웃 이전 채팅 로딩 정상
[ ] DM 채널 장시간 사용 및 unread dot은 별도 전체 회귀 테스트에서 재확인

4. Token 권한 테스트
-------------------
[x] GM은 캐릭터 토큰/패널 토큰 생성/이동/편집/삭제 가능
[x] 권한 없는 플레이어의 위험 조작은 Rules 단계에서 차단됨
[ ] createToken 권한 플레이어 생성 가능 테스트는 권한 UI 구현 후 확인
[ ] moveToken 권한 플레이어 이동 가능 테스트는 권한 UI 구현 후 확인
[ ] editToken 권한 플레이어 편집/삭제 가능 테스트는 권한 UI 구현 후 확인
[ ] 권한 회수 후 기능 차단은 권한 UI 구현 후 확인

5. 맵세팅 / 장면 전환
--------------------
[x] GM 맵세팅 ZIP 업로드 가능
[x] 맵세팅 레이어 숨김/표시/순서/삭제 정상
[x] 전체 삭제 정상
[x] mapScenes 저장/전환/복제/순서 변경 정상
[x] 위치고정 imported panel token 위 드래그는 맵 팬 유지
[ ] manageMap 권한 플레이어 테스트는 권한 UI 구현 후 확인

6. BGM 권한
-----------
[보류] GM은 BGM 추가/삭제/재생/이전/다음 가능
[보류] manageBgm 권한 플레이어는 BGM 수정 가능
[보류] 권한 없는 플레이어는 BGM 수정 불가

메모:
- 현재 BGM 기능이 실제 사이트에서 아직 완성/사용 상태가 아니므로 manageBgm 테스트는 보류한다.
- 단, Rules에서는 BGM 권한과 맵세팅 권한을 분리해 둔 상태다.

7. Journals / Handouts
----------------------
[x] 기존 저널/핸드아웃 기능이 Rules 적용 후 깨지지 않음
[ ] journals read per-item 하드닝은 아직 하지 않음
[ ] handouts read per-item 하드닝은 아직 하지 않음
[ ] 저널/핸드아웃 로딩 구조를 바꾸는 별도 로드맵 필요

8. 롤백 기준
------------
아래 중 하나라도 발생하면 `database.rules.rollback-current.json`을 Firebase Console에 다시 붙여넣고 Publish한다.

- 방 생성/입장 실패
- GM 방 삭제 실패
- 일반 채팅/잡담/DM 전송 실패
- GM 토큰 생성/이동/편집 실패
- GM 맵세팅 ZIP 업로드 Permission denied
- GM 장면 전환 저장/복제/순서 변경 Permission denied
- 저널/핸드아웃 저장 실패
- 예상하지 못한 PERMISSION_DENIED 반복

9. 다음 권장 작업
----------------
1. 토큰 권한 UI 구현 여부 검토
2. 권한 없는 플레이어 버튼 숨김/비활성화 UX 정리
3. journals/handouts 권한 구조 별도 로드맵 작성
4. 최종 통합 회귀 테스트

저널 / 핸드아웃 권한 1차 추가 확인
---------------------------------
- [ ] GM이 저널 권한 저장 시 `assignedTo`와 `assignedMap`이 함께 저장되는지 확인한다.
- [ ] 권한 받은 플레이어가 저널을 열람/수정할 수 있는지 확인한다.
- [ ] 권한 받은 플레이어가 저널 삭제를 할 수 없는지 확인한다.
- [ ] 권한 받은 플레이어가 `assignedTo` / `assignedMap`을 직접 바꿀 수 없는지 확인한다.
- [ ] GM이 핸드아웃 저장 시 `allowedTo`와 `allowedMap`이 함께 저장되는지 확인한다.
- [ ] 플레이어가 허용된 핸드아웃만 열람하고, 수정/삭제 UI는 보이지 않는지 확인한다.

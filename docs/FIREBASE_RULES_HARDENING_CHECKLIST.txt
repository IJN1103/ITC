Firebase Rules 하드닝 체크리스트
================================

0. 전제
-------
- 이 체크리스트는 Firebase Console Rules를 실제로 바꾸기 전에 사용한다.
- `database.rules.json`은 현재 콘솔 Rules 스냅샷이다.
- `database.rules.hardening-draft.json`은 검토용 초안이며, 바로 운영 콘솔에 붙여넣지 않는다.

1. 현재 취약점 확인
------------------
[ ] 일반 플레이어가 rooms/{roomCode} 삭제 조건에 걸릴 수 있는지 확인
[ ] players 상위 `.write: auth != null` 제거 필요성 확인
[ ] bgm/tokens/journals/handouts 참가자 전체 write 위험 확인
[ ] dmChats 경로가 Rules에 명시되어 있는지 확인

2. 클라이언트 방어 코드 확인
-------------------------
[ ] 일반 플레이어가 마지막으로 방을 나가도 rooms/{roomCode}를 삭제하지 않는다.
[ ] GM/owner의 방 종료 버튼은 기존처럼 방 전체 삭제를 수행한다.
[ ] 로비 내 GM 세션 삭제는 기존처럼 방 전체 삭제를 수행한다.
[ ] 일반 플레이어의 로비 X 버튼은 내 세션 목록에서만 제거한다.

3. BGM 권한 테스트
-----------------
[ ] GM은 BGM 추가/삭제/재생/이전/다음 가능
[ ] 권한 없는 플레이어는 BGM 수정 불가
[ ] GM이 manageBgm 권한 부여
[ ] 권한 받은 플레이어는 BGM 수정 가능
[ ] 권한 회수 후 다시 수정 불가

4. Token 권한 테스트
-------------------
[ ] GM은 캐릭터 토큰/패널 토큰 생성 가능
[ ] 권한 없는 플레이어는 토큰 생성 불가
[ ] createToken 권한 플레이어는 생성 가능
[ ] moveToken 권한 플레이어는 이동 가능
[ ] editToken 권한 플레이어는 편집/삭제 가능
[ ] 권한 회수 후 기능 차단
[ ] 맵세팅 ZIP imported panel token은 GM 기능으로 정상 생성
[ ] 위치고정 imported panel token 위 드래그는 맵 팬 유지

5. Journals / Handouts 테스트
----------------------------
[ ] 현재 코드가 journals 전체 로딩 후 프론트 필터링을 쓰는 점 확인
[ ] 현재 코드가 handouts 전체 로딩 후 프론트 필터링을 쓰는 점 확인
[ ] Rules에서 read를 바로 엄격하게 걸지 않는다
[ ] write만 1차 제한할 경우 기존 저장/편집이 깨지지 않는지 확인
[ ] 저널 권한 assignedTo 기준 확인
[ ] 핸드아웃 allowedTo 기준 확인

6. Chat / DM / Popout 회귀 테스트
--------------------------------
[ ] 일반 채팅 전송 가능
[ ] desc 지문 기본창/팝아웃 동일 스타일
[ ] 이미지가 팝아웃에서 URL 텍스트로 보이지 않음
[ ] speak-as 이미지 정상
[ ] @스탠딩이름 화면 표시 숨김
[ ] DM 채널 전송/수신 정상
[ ] dmChats 경로를 쓰는 경우 권한 오류 없음
[ ] 기본창/팝아웃 이전 채팅 로딩 정상

7. 장면 전환 / 맵세팅 회귀 테스트
--------------------------------
[ ] 씬 전환 정상
[ ] 씬 복제 정상
[ ] 씬 카드 드래그 순서 저장 정상
[ ] mapScenes GM write 정상
[ ] activeSceneId meta write 정상
[ ] 맵세팅 ZIP 업로드 정상
[ ] 레이어 숨김/표시/순서/삭제 정상
[ ] 전체 삭제 정상

8. 적용 여부 판단
----------------
[ ] 테스트 방에서 2계정 이상으로 검증 완료
[ ] Permission denied 발생 위치 없음
[ ] 콘솔 Rules 적용 후 새로고침/재입장 테스트 완료
[ ] 문제가 있으면 즉시 `database.rules.json` 기준으로 롤백

9. Stage 2 추가 테스트
---------------------
[ ] 일반 플레이어가 마지막으로 방에서 나가도 rooms/{roomCode}가 삭제되지 않음
[ ] GM/owner는 방 종료 또는 로비 삭제로 rooms/{roomCode} 삭제 가능
[ ] 신규 플레이어가 방 코드로 입장 가능
[ ] 기존 플레이어가 재입장 가능
[ ] 플레이어가 자기 role을 gm으로 바꿀 수 없음
[ ] 플레이어가 자기 permissions를 직접 추가/수정할 수 없음
[ ] 권한 없는 플레이어가 BGM 트랙 클릭/삭제/재생 변경을 할 수 없음
[ ] manageBgm 권한 플레이어는 BGM 수정 가능
[ ] 권한 없는 플레이어도 GM이 바꾼 BGM 재생 상태는 정상 수신
[ ] moveToken 권한은 토큰 x/y 이동만 가능
[ ] editToken 권한이 있어야 토큰 편집/삭제 가능
[ ] dmChats meta 생성은 GM/owner만 가능

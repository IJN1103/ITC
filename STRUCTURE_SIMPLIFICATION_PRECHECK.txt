TRPG 웹사이트 다음 단계 사전 점검표
=================================

현재 기준
---------
- base64 제거 작업: 마감 단계
- 업로드 UX 안정화: 마감 단계
- 다음 큰 작업: 구조 단순화 전 사전 점검 + 성능 최적화 준비

이번 문서는 '바로 큰 리팩토링에 들어가기 전에' 확인해야 할 것만 정리한 체크리스트입니다.

1. 지금 당장 건드리지 말아야 하는 핵심 UX
------------------------------------
아래 항목은 다음 리팩토링에서도 반드시 유지해야 합니다.

[채팅 이미지]
- + 버튼으로 이미지 선택
- 최대 4장까지 선택
- 선택 직후 바로 전송되지 않고 미리보기 큐에 올라갈 것
- 미리보기에서 X 버튼으로 개별 삭제 가능할 것
- 드래그로 순서 변경 가능할 것
- 전송 버튼으로만 실제 전송될 것
- 가로폭 채우기 옵션 유지
- 이미지 1장마다 개별 메시지로 전송될 것

[프로필]
- 크롭 후 저장
- 저장 중 중복 클릭 방지
- 실패 시 재시도 가능 상태 유지

[채팅 구조]
- 최신 메시지 append-only
- 일정 개수 초과 시 위쪽 DOM 제거
- 위로 스크롤 시 과거 메시지 복원

2. 다음 리팩토링 전 필수 테스트
----------------------------
아래 테스트를 먼저 통과한 뒤 구조 단순화 작업으로 넘어가는 것을 권장합니다.

[프로필]
- 프로필 사진 변경 후 즉시 반영
- 새로고침 후 유지
- 재로그인 후 유지
- 실패 메시지 정상 표시

[채팅 이미지]
- 일반 JPG 1장 전송
- PNG 1장 전송
- GIF 1장 전송
- 4장 동시 전송
- 준비 중 잠금 확인
- 업로드 중 잠금 확인
- 중간 실패 시 남은 이미지 큐 복구 확인

[아바타]
- 일반 채팅 아바타
- 잡담 탭 아바타
- 팝아웃 아바타
- speak-as 아바타
- 저널 아바타
- 플레이어 목록 아바타

3. 다음 단계에서 가장 먼저 단순화할 후보
------------------------------------
우선순위 A
- chat.js 안의 이미지 준비/전송 상태 관리 보조 함수 정리
- profile.js 안의 업로드/동기화/세션 정리 함수 정리

우선순위 B
- localStorage avatar 관련 접근 경로 공통 helper로 묶기
- sanitize 계열 함수 중 중복되는 것 정리

우선순위 C
- speak-as / popout / journal / lobby / game 쪽 avatar 조회 공통화

4. 다음 단계에서 '아직' 하지 말아야 할 것
------------------------------------
- 강한 가상스크롤 재도입
- 채팅 렌더링 구조 대규모 교체
- 메시지 저장 구조 전체 교체
- 이미지 시스템 전면 재작성
- Firebase 구조 변경

5. 구조 단순화 들어가기 전 권장 순서
--------------------------------
1) helper 공통화 후보만 찾기
2) 중복 함수 제거
3) avatar 조회 공통화
4) 업로드 상태 UI 공통화
5) 그 다음에만 성능 미세 최적화

6. 완료 기준
----------
아래가 되면 다음 단계 성공으로 봅니다.
- 기능 변화 없음
- 파일 수 최소 변경
- 기존 UX 그대로 유지
- 코드 중복 감소
- 테스트 항목 모두 통과
